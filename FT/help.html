Help for Family Tree


Introduction

This is a simple free app to view family trees.  The input is a comma-separated file (as described later) and the output is an interactive family tree. 


Input file

The input is a csv (comma-separated file) with the multiple rows. A csv file can be easily created using a spreadsheet app and saving the file as csv.

Each individual or union is recoded in a separate row. The row can have one or two individuals as describes below.

  • An individual will have a name and one or many options in brackets. The name can be words separated by space.
  • Each option is a key followed by space followed by a word:
    • key = aka followed by a word (other name); aka stands for also known as.
    • key = gender followed by f or m for female and male respectively. If no gender is provided male is assumed
  • If there is a spouse, then it is recorded with a + sign. Examples in table below.
  • When a spouse is present, best practice suggests that the first individual must be the direct descendant of the family.

    • Examples of individuals, unions and options

    Written as Means
    xxx (gender f) Individual with name = xxx, gender = female
    xxx xyz(gender m) Individual with name = xxx xyz, gender = male. If no gender is provided it defaults to male
    xxx Individual with name = xxx, gender = male
    xxx (aka yyy) Individual with name = xxx, aka = yyy, gender = male
    xxx (aka yyy gender f) Individual with name = xxx, aka = yyy, gender = female
    xxx (gender f aka yyy) Individual with name = xxx, aka = yyy, gender = female
    xxx xyz + yyy (gender f) Direct descendant's name = xxx xyz, gender = male and spouse with name = yyy, gender = female
    xxx (gender f) + yyy (aka zzz) Direct descendant's name = xxx, gender = female and spouse with name = yyy, aka = zzz, gender = male

If the first letter in the first column of a row is a # it means it is a directive to the app rather than an individual of union.

  • #title means app will display the title with the words following the directive. if is not provided the app uses the first descendant's name as default.
  • #note means the app will display note with the words following the directive. Multiple notes are allowed

The first individual or union from the first generation is written in column 1. The children are written in rows below the parents, one row for a child, and in column N+1 where the parents are in column N. Best practice suggests that childen are recorded oldest child is one the very next row after parents.

Output

The family tree is initailly displayed as "left to right". The view can be changed to "top down" from the menu.

The hamberger menu is on the top left. The menu options are:

  • Load file: allows a new file to be loaded.
  • Left right view: displays tree "left to right".
  • Top down view: displays tree "top down".
  • Example: displays an sample tree
  • Help: displays help (this page).
